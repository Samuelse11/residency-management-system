# Professional Residency Management System (SGRP)

A full-stack web application designed to digitize and streamline the professional residency process by replacing paper-based workflows with a centralized platform for students, advisors, coordinators, and administrators.

# Overview

The Professional Residency Management System (SGRP) was developed to modernize the university's professional residency process. Before this project, the entire workflow relied heavily on paper documentation and email communication, making it difficult to manage applications, review documents, and monitor student progress efficiently.

SGRP centralizes every stage of the residency process into a single platform where users can submit documentation, monitor application progress, communicate internally, and manage residency decisions through role-based workflows.

# Problem Statement

The traditional residency process presented several operational challenges:

- Paper-based documentation increased administrative workload.
- Communication was fragmented across multiple email threads.
- Student progress was difficult to monitor.
- Reviewing documentation required manual processes.
- Important information could easily be misplaced.
- There was no centralized platform to manage the complete workflow.

#  Solution

SGRP provides a centralized web platform that digitizes the entire professional residency process.

The system enables students, advisors, coordinators, and administrators to collaborate within a single application, improving communication, document management, workflow visibility, and residency tracking while significantly reducing manual administrative tasks.

#  Project Impact

The platform transformed a manual administrative process into a centralized digital solution by:

- Reducing dependency on physical documentation.
- Centralizing communication between students and university staff.
- Improving visibility throughout the residency workflow.
- Simplifying document submission and review.
- Providing structured tracking of residency applications.
- Increasing process transparency for all participants.

#  Database Design

The application uses a relational MySQL database designed to support the complete residency workflow.

The database manages:

- Students
- Advisors
- Coordinators
- Administrators
- Residency applications
- Uploaded documents
- User roles
- Residency status
- Comments and observations

The schema was normalized to reduce redundancy while maintaining data integrity and supporting efficient queries.
